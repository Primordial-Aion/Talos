TASK 1 — ENGINE CLEANUP

Goal
Make all engine resources properly released.

Current issue:
Engine only destroys window.
GPU resources likely leak.

Files to inspect
- engine/core/Engine.java
- engine/scene/Scene.java
- engine/ui/UIManager.java
- any class with cleanup(), destroy(), or dispose()

Required changes

Step 1
Open Engine.run().
Find final cleanup call.
Current behavior is window only.
Replace with full shutdown pipeline.

Engine cleanup must call, in a sensible order:
- Scene.get().cleanup()
- UIManager.get().cleanup()
- Renderer.get().cleanup()
- ShaderProgram.cleanupAll()
- AudioManager.get().cleanup()
- Window.get().cleanup()

If any call does not exist, create the missing method.
If a subsystem has no real cleanup yet, implement it.

Step 2
Open Scene.cleanup().
Current behavior likely only clears lists.
That is wrong.

Before clearing:
- iterate all entities
- call cleanup on each entity or its owned resources
- free models, meshes, textures, shaders, and other GPU objects if owned there
- clear references only after cleanup is done

Step 3
Open UIManager.cleanup().
Current behavior likely only cleans top-level UI elements.
That is incomplete.

Fix it so cleanup walks nested children too.
If panels contain child elements, recursively clean them.
Do not leave nested UI resources alive.

Step 4
After cleanup is fixed, trace the shutdown path.
Confirm that everything owned by the scene and UI gets a release path.

TASK 2 — INPUT STATE

Goal

Files to inspect

Required changes

Step 4

TASK 3 — RENDER HOT PATH ALLOCATION

Files to inspect
- engine/scene/Scene.java
- engine/entity/Entity.java
- engine/model/Model.java
- engine/ui/UIElement.java
- engine/shader/ShaderProgram.java

Required changes

Step 1
Find every place creating new float[16] inside render paths.
Remove repeated array allocation from per-frame drawing code.

Use reusable buffers or cached arrays where appropriate.

Step 2

Step 3
Make sure hot paths do not do repeated expensive work.

Step 4
After changes, trace one frame render path.
Confirm matrix data reaches GPU without new allocation spam.
